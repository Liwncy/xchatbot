# 修仙插件迭代方案（专用记录）

> 目标：在当前 `xchatbot` 的修仙插件基础上，按周渐进补齐玩法，保证每周都可上线、可回滚、可验证。

## 0. 当前基线（已完成）

当前插件目录：`src/plugins/game/xiuxian/`

已具备：

- 统一四字指令体系（如：`修仙修炼`、`修仙探索`）
- 基础循环：创建 / 状态 / 修炼 / 探索 / 背包 / 装备 / 卸装 / 挑战
- 战报能力：`修仙战报`、`修仙战详`
- 背包增强：筛选（类型/品质）+ 排序（评分升降/最新）
- 未识别修仙命令提示
- 状态面板增强（战力细项、背包占用、装备栏）

## 2. 四周迭代计划（每周一个可交付）

## 第1周：体验补全（进行中）

目标：现有系统“可查、可读、可引导”。

范围：

- [x] 战报查询
- [x] 背包筛选/排序
- [x] 状态面板美化
- [x] 未知指令引导
- [ ] 体验回归与文案微调

验收标准：

- `修仙战报 [页码]` 可正常分页
- `修仙战详 [id]` 可返回战斗摘要
- `修仙背包 1 神兵 评分降序` 可用
- 未知 `修仙` 前缀命令不会静默

## 第2周：经济闭环

目标：形成“产出-消耗-回收”链路。

范围：

- [x] `修仙商店`（查看刷新池）
- [x] `修仙购买 [商品ID]`
- [x] `修仙出售 [装备ID]`
- [x] 经济流水查询（最近 N 条）

建议新增数据表：

- `xiuxian_shop_offers`
- `xiuxian_economy_logs`

验收标准：

- 购买与出售都能完整扣加灵石
- 不出现负余额与重复扣款
- 流水可追溯每次变动

## 第3周：成长目标

目标：建立短中期目标与回流动机。

范围：

- [x] `修仙签到`
- [x] `修仙任务` / `修仙领奖`
- [x] `修仙成就`

建议新增数据表：

- `xiuxian_checkins`
- `xiuxian_tasks`
- `xiuxian_player_tasks`
- `xiuxian_achievements`
- `xiuxian_player_achievements`

验收标准：

- 每日只能签到一次
- 任务进度可累计并领取奖励
- 成就达成触发一次性奖励

## 第4周：高光玩法（BOSS）

目标：增加阶段性强玩法。

范围：

- [x] `修仙讨伐`
- [x] BOSS 结算与掉落
- [x] BOSS 战报展示
- [x] 多人共享BOSS（同会话共享血量与贡献榜）

建议新增数据表：

- `xiuxian_boss_states`
- `xiuxian_boss_logs`

验收标准：

- 可发起讨伐并计算胜负
- 掉落进入背包可用
- 战报可回溯

## 4. 风险与规避

- 并发写覆盖：后续给玩家写入引入乐观锁（`version`）与重试
- 半成功状态：关键动作尽量批处理，避免奖励成功但冷却失败
- 经济失衡：上线第2周时先保守定价，结合流水再调参
- 消息过长：统一分页输出，单次文本控制在可读范围

按计划继续推进：

1. 完成第1周收尾（文案与回归）

4. 进入下一阶段：规划 `NPC/情缘/宠物` 或 `爬塔排行`
